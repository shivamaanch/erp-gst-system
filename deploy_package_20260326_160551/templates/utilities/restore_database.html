Restore Database

Critical Warning

This will replace your entire database! All current data will be lost and replaced with the backup file. Make sure you have a recent backup before proceeding.

Restore Process:
  1. Your current database will be automatically backed up before restore
  2. The uploaded backup file will replace the current database
  3. The application will use the restored data immediately
  4. You may need to log in again after restore
Only .db files are accepted. File should be a valid ERP database backup.
Safety Features
  • Current database will be backed up before restore
  • Backup saved as: erp_before_restore_YYYYMMDD_HHMMSS.db
  • You can restore the previous version if needed
  • File validation ensures only .db files are accepted

Important Notes
  • Backup First: Always create a backup before restoring
  • Valid File: Ensure the backup file is a valid ERP database
  • File Size: Large files may take time to upload
  • Re-login: You may need to log in again after restore
  • Test Environment: Test restores in a test environment first if possible
  • Data Loss: All current data will be replaced - this cannot be undone
When to Restore
  • After accidental data deletion
  • To recover from data corruption
  • To revert changes made by utilities
  • To restore to a previous state
  • To migrate data from another system
